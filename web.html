Introduction

Welcome to the documentation for DevPortfolio, a responsive and customizable portfolio template for developers and designers. This documentation will guide you through the setup, customization, and features of your portfolio website.

Note: This documentation assumes basic knowledge of HTML, CSS, and JavaScript. If you're new to web development, we recommend checking out resources like MDN Web Docs or W3Schools to learn the basics.

Getting Started

To get started with your DevPortfolio, follow these simple steps:

Prerequisites

  • A text editor (VS Code, Sublime Text, Atom, etc.)
  • Basic knowledge of HTML, CSS, and JavaScript
  • A web browser for testing

Installation

  1. Download the portfolio template files
  2. Extract the files to your preferred location
  3. Open the files in your text editor
  4. Start customizing!

Quick Start

The quickest way to get started is to edit the index.html file and replace the placeholder content with your own information. Key areas to update include:

  • Personal information in the hero section
  • Your skills and proficiency levels
  • Project details and images
  • Contact information

File Structure

Understanding the file structure will help you navigate and customize your portfolio efficiently:

├── index.html # Main homepage ├── documentation.html # This documentation page ├── project-details.html # Template for project details ├── css/ │ └── styles.css # Main CSS file (included in HTML) ├── js/ │ └── main.js # Main JavaScript file └── images/ # Directory for your images

Key Files

  • index.html - The main portfolio page with all sections (hero, about, skills, projects, contact)
  • documentation.html - This documentation page
  • project-details.html - Template for individual project pages
Tip: Keep your file structure organized. Place all images in the 'images' folder and maintain a logical naming convention for your files.

Customization

Your DevPortfolio is highly customizable. Here are the main areas you can personalize:

Color Scheme

The color scheme is defined using CSS variables. To change the colors, locate the :root selector in the CSS and modify the color values:

:root { --primary: #6366f1; /* Primary color */ --secondary: #8b5cf6; /* Secondary color */ --accent: #ec4899; /* Accent color */ --dark: #1e293b; /* Dark text color */ --light: #f8fafc; /* Light background color */ --background: #f1f5f9; /* Main background color */ }

Typography

The portfolio uses the system font stack by default. To change the font family, modify the font-family property in the CSS file:

* { font-family: 'Your-Font', sans-serif; }

Adding Your Own Content

  • Personal Information: Update the hero section with your name, professional title, and brief introduction.
  • About Section: Replace the placeholder text with your own bio and professional background.
  • Skills: Add or remove skills in the skills section and adjust the proficiency percentages.
  • Projects: Update the project cards with your own projects, descriptions, and tags.
  • Contact Information: Replace the placeholder contact details with your actual information.
Important: When adding your own images, make sure they are optimized for web to ensure fast loading times. Replace the placeholder images with your own by changing the src attributes in the image tags.

Responsive Design

The DevPortfolio is fully responsive and works on all devices. Here's how the responsive design works:

Breakpoints

The main breakpoint is set at 768px for mobile devices. Below this width, the layout changes to a mobile-friendly design.

@media (max-width: 768px) { /* Mobile responsive styles */ }

Mobile Menu

On mobile devices, the navigation menu transforms into a hamburger menu. The JavaScript in main.js handles the toggling of the mobile menu.

Fluid Layout

The portfolio uses a fluid layout with percentage-based widths and flexible grid systems to ensure content looks good on all screen sizes.

Testing Responsive Design

To test how your portfolio looks on different devices:

  1. Use your browser's developer tools (F12 or right-click and select "Inspect")
  2. Toggle the device toolbar (usually an icon showing a mobile and tablet)
  3. Select different device presets or adjust the width manually

JavaScript Features

The portfolio includes several JavaScript features to enhance user experience:

Mobile Menu Toggle

JavaScript is used to toggle the mobile menu when the hamburger icon is clicked:

// Mobile Menu Toggle const menuToggle = document.getElementById('menuToggle'); const navLinks = document.getElementById('navLinks'); if (menuToggle) { menuToggle.addEventListener('click', () => { navLinks.classList.toggle('active'); }); }

Smooth Scrolling

For a better user experience, smooth scrolling is implemented for navigation links:

// Smooth scrolling for navigation links document.querySelectorAll('a[href^="#"]').forEach(anchor => { anchor.addEventListener('click', function (e) { e.preventDefault(); document.querySelector(this.getAttribute('href')).scrollIntoView({ behavior: 'smooth' }); }); });

Form Validation

Basic form validation is included for the contact form:

// Form validation const contactForm = document.getElementById('contactForm'); if (contactForm) { contactForm.addEventListener('submit', function(e) { e.preventDefault(); // Basic validation const name = document.getElementById('name').value; const email = document.getElementById('email').value; const message = document.getElementById('message').value; if (!name || !email || !message) { alert('Please fill in all fields'); return; } // You would normally send the form data to a server here alert('Message sent successfully!'); contactForm.reset(); }); }

Performance Tips

To ensure your portfolio loads quickly and performs well:

Image Optimization

  • Optimize all images before uploading (use tools like TinyPNG or Squoosh)
  • Use appropriate image formats (JPEG for photos, PNG for graphics with transparency, SVG for icons)
  • Specify width and height attributes on images to prevent layout shifts

Code Optimization

  • Minify your CSS and JavaScript files for production
  • Remove any unused CSS or JavaScript
  • Combine multiple CSS or JavaScript files to reduce HTTP requests

Loading Strategies

  • Place critical CSS inline in the head
  • Use defer attribute for non-critical JavaScript
  • Lazy load images that are below the fold
Tip: Use tools like Google PageSpeed Insights or Lighthouse to analyze and improve your portfolio's performance.

SEO Optimization

To improve your portfolio's visibility in search engines:

Meta Tags

Update the meta tags in the head section of each HTML file:

Your Name - Portfolio

Semantic HTML

Use semantic HTML elements to help search engines understand your content:

  • Use <header>, <nav>, <main>, <section>, <article>, <footer> instead of just <div>
  • Use appropriate heading levels (<h1> through <h6>) in a hierarchical order
  • Include alt text for all images

Troubleshooting

Common issues and their solutions:

Layout Issues

Problem Solution
Content overflows on mobile Check if you've properly set box-sizing: border-box and ensure all containers have max-width and proper padding
Elements not aligning properly Inspect elements using browser dev tools and check CSS rules affecting alignment
Images distorted Ensure you're using proper aspect ratios and object-fit properties

JavaScript Issues

Problem Solution
Mobile menu not working Check if IDs match between HTML and JavaScript, and ensure event listeners are properly attached
Smooth scrolling not working Verify that your anchor links have correct IDs that match the target elements
Form submission errors Check browser console for JavaScript errors and ensure form fields have proper validation
